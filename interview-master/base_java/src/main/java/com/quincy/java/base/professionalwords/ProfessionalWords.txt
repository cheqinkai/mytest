  并发量的计算公式
   不考虑任何客观的因素，假设一个服务器有2个物理CPU,每个CPU有8核16个线程，那么它的1s极限并发量是
   1000ms*16(CPU线程数量)*2个 / (CPU切片轮换时间(假设10ms) + 程序执行时间(假设10ms)) ＝ 1600个并发
   如果前台保证不死机的情况下，假设用户可容忍的最大时间是3s,那么这套程序的最大并发就是4800个，当然这是
   一个理想的状况，实际上受其他因素的影响，肯定要比这个少
